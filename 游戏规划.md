# 宝可梦文字对战游戏 - 游戏规划

## 🎮 游戏是什么样的？

这是一个**纯文字**的回合制对战游戏，就像下棋一样，你和电脑轮流出招。

### 游戏画面示例：
```
==========================================
你的宝可梦：皮卡丘
生命值：❤️❤️❤️❤️❤️ (35/35)

VS

对手宝可梦：小火龙
生命值：❤️❤️❤️🖤🖤 (23/39)
==========================================

轮到你了！选择你的行动：
1. 十万伏特 (电系，威力90)
2. 电光一闪 (普通系，威力40)
3. 使用药水
4. 更换宝可梦

请输入数字 (1-4)：
```

---

## 🧱 游戏的主要组成部分（像搭积木）

### 第1块积木：宝可梦数据
每个宝可梦就像一张卡片，上面有：
- **名字**：比如皮卡丘、小火龙
- **属性**：电系、火系、水系等
- **生命值 (HP)**：比如35点，打到0就输了
- **攻击力**：比如55，决定打人多痛
- **防御力**：比如40，决定被打多痛
- **速度**：比如90，决定谁先出手
- **技能**：4个招式，比如"十万伏特"

**简单理解**：就像游戏王卡片，每张卡有不同的数值和技能。

---

### 第2块积木：属性相克系统
就像"石头剪刀布"：
- 💧 **水克火**：水系打火系，伤害 ×2
- 🔥 **火克草**：火系打草系，伤害 ×2
- 🌿 **草克水**：草系打水系，伤害 ×2
- ⚡ **电克水**：电系打水系，伤害 ×2
- 🪨 **地面克电**：地面打电系，伤害 ×2

**简单理解**：带对属性的宝可梦，就像带了"法术加成"。

---

### 第3块积木：战斗系统
这是游戏的核心，就像下回合制棋：

#### 每个回合的流程：
1. **你选择行动**（攻击/换宝可梦/用道具）
2. **电脑AI选择行动**
3. **比较速度**，快的先攻击
4. **计算伤害**
5. **检查输赢**：有一方的宝可梦倒下了吗？

#### 伤害计算公式（不用理解细节）：
```
基础伤害 = (攻击力 × 技能威力) ÷ 防御力
最终伤害 = 基础伤害 × 属性克制倍数 × 随机数(0.85~1.0)
```

**简单理解**：就像打架，你的力气、武器、对方的防护、还有运气，都影响伤害。

---

### 第4块积木：玩家背包系统
你可以携带6只宝可梦和一些道具：
- **宝可梦队伍**：最多6只，战斗中可以换
- **道具**：
  - 🍶 **普通药水**：恢复20点生命
  - 🍾 **高级药水**：恢复50点生命
  - ⚡ **能量饮料**：恢复所有生命

**简单理解**：就像背包里装着6个怪兽球和一些补给品。

---

### 第5块积木：AI对手
电脑控制的对手，会自动做决策：
- **简单AI**：随机选技能
- **中等AI**：会选择克制你的技能
- **困难AI**：会根据血量决定是否换宝可梦或使用道具

**简单理解**：就像和电脑下棋，电脑会"思考"怎么赢你。

---

## 🎯 游戏模式

### 模式1：单场对战
- 你和电脑各选3只宝可梦
- 打到一方全部倒下为止
- 适合快速玩一局

### 模式2：道馆挑战（可选）
- 挑战8个道馆馆主
- 每个馆主越来越强
- 打败全部后成为冠军

## 📁 游戏文件结构（技术部分）

```
pokemon-game/
├── game.py              # 主程序，启动游戏
├── pokemon.py           # 宝可梦的"设计图"
├── battle.py            # 战斗系统的"规则"
├── player.py            # 玩家背包和队伍
├── ai.py                # 电脑对手的"大脑"
├── data/
│   ├── pokemons.json    # 所有宝可梦的数据
│   └── moves.json       # 所有技能的数据
└── README.md            # 使用说明
```

**简单理解**：
- `.py` 文件 = 程序代码（像游戏的说明书）
- `.json` 文件 = 数据文件（像游戏的数据库）

## 💡 技术选择（给开发者看的）

### 编程语言：Python
**为什么选Python？**
- 语法简单，容易上手
- 纯文字游戏不需要复杂的图形库
- 标准库就能完成所有功能

### 核心技术：
- **面向对象编程**：用类(Class)表示宝可梦、技能、玩家
- **JSON数据存储**：用JSON文件存储宝可梦和技能数据
- **命令行界面**：用 `input()` 和 `print()` 实现交互

### 不需要的技术：
- ❌ 不需要数据库（用JSON文件就够了）
- ❌ 不需要图形界面（纯文字）
- ❌ 不需要网络功能（单机游戏）

## 🎓 总结：这个游戏的本质

把复杂的概念用最简单的话说：

1. **宝可梦** = 有不同数值的角色卡片
2. **战斗** = 回合制下棋，比谁的数值和策略更好
3. **属性相克** = 石头剪刀布的加强版
4. **AI** = 会自动选择的电脑对手
5. **整个游戏** = 一个会计算数学、比较大小、按规则运行的程序

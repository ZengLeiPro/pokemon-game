## ℹ️ 为什么归档？

### 问题
- 约 30-40% 的内容重复（项目结构、游戏玩法等）
- 文档职责不清，维护成本高
- 新开发者需要阅读 4000+ 行才能全面了解

### 解决方案
- 将 7 个文档精简为 3 个核心文档
- 减少 40-45% 的冗余内容（约 1700-1900 行）
- 信息损失率仅 5-10%（主要是重复和过时内容）
